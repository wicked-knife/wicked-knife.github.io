ES6中最实用的特性

记录几个ES6中最实用的几个特性,可以很方便地运用到平时开发中。

0.目录

  • 函数参数默认值
  • 模板字符串
  • 解构赋值
  • 对象属性简写
  • 箭头函数
  • Promise
  • Let与Const
  • 类
  • 模块化
  • Set

1.函数参数默认值

非ES6:


function foo(height, color)
{
    var height = height || 50
    var color = color || 'red'
    //...
}

这样写一般没什么问题，但是当我们这样调用foo函数时：



foo(0)

0将进行强制类型转换为false,于是height就成了50。这显然不是我们所期望的。

ES6：


function foo(height = 50, color = 'red')
{
    // ...
}

上面这种情况就能得到很好地解决。

2.模板字符串

非ES6:


var name = 'Your name is ' + first + ' ' + last + '.'

只能使用+来进行字符串的拼接,这样不仅麻烦,而且在拼接的内容较复杂时,可读性会变得非常差,最重要的是非常容易出错!

ES6：


var name = `Your name is ${first} ${last}.`

变量直接放在${}中,代码简洁,直观。

3.解构赋值

非ES6:

如获取对象属性值时:


var data = $('body').data()  // data有house和mouse属性
var house = data.house
var mouse = data.mouse

ES6:

获取对象属性值时:


var { house, mouse} = $('body').data()

对于数组:


var [col1, col2]  = $('.column')

4.对象属性简写

非ES6:

当对象的key值和value的引用名称相同时:


var bar = 'bar'
var foo = function ()
{
    // ...
}
var baz = {
  bar: bar,
  foo: foo
}

ES6:

对象中直接写变量，非常简单：


var bar = 'bar'
var foo = function ()
{
    // ...
}
var baz = { bar, foo }

5.箭头函数

非ES6:

普通函数体内的this, 指向调用该函数的对象。


function foo() 
{
    console.log(this.id)
}
var id = 1
foo() // 输出1
foo.call({ id: 2 }) // 输出2

ES6:

箭头函数内的this，就是定义该函数时this所指向的对象，而不是调用时所在的对象。


var foo = () => {
  console.log(this.id)
}
var id = 1
foo() // 输出1
foo.call({ id: 2 }) // 输出1

6.Promise

非ES6:

嵌套两个setTimeout回调函数：


setTimeout(function()
{
    console.log('Hello') // 1秒后输出"Hello"
    setTimeout(function()
    {
        console.log('Fundebug') // 2秒后输出"Fundebug"
    }, 1000)
}, 1000)

ES6:

使用两个then是异步编程串行化，避免了回调嵌套回调：


var wait1000 = new Promise(function(resolve, reject)
{
    setTimeout(resolve, 1000)
})
wait1000
    .then(function()
    {
        console.log("Hello") // 1秒后输出"Hello"
        return wait1000
    })
    .then(function()
    {
        console.log("Fundebug") // 2秒后输出"Fundebug"
    })

7.Let与Const

非ES6:

var定义的变量没有块级作用域：


{
  var a = 10
}
console.log(a) // 输出10

ES6:

let定义的变量为块级作用域，因此会报错：


{
  let a = 10
  const b = 10
}
console.log(a) // “ReferenceError: a is not defined”
console.log(b) // “ReferenceError: b is not defined”

8.Class

非ES6:

使用构造函数创建对象：


function Point(x, y)
{
    this.x = x
    this.y = y
    this.add = function()
    {
        return this.x + this.y
    }
}
var p = new Point(1, 2)
console.log(p.add()) // 输出3

ES6:

使用Class定义类，更加规范，继承实现起来也很方便：


class Point
{
    constructor(x, y)
    {
        this.x = x
        this.y = y
    }
    add()
    {
        return this.x + this.y
    }
}
var p = new Point(1, 2)
console.log(p.add()) // 输出3

9.模块化

JavaScript一直没有官方的模块化解决方案，开发者在实践中主要采用CommonJS和AMD规范。而ES6制定了模块(Module)功能。

非ES6:

Node.js采用CommenJS规范实现了模块化，而前端也可以采用，只是在部署时需要使用Browserify等工具打包。这里不妨介绍一下CommenJS规范。

module.js中使用module.exports导出port变量和getAccounts函数：


// module.js
module.exports = {
  port: 3000,
  getAccounts: function() {
    ...
  }
}

main.js中使用require导入module.js：


// main.js
var service = require('module.js')
console.log(service.port) // 输出3000

ES6:

ES6中使用export与import关键字实现模块化。

module.js中使用export导出port变量和getAccounts函数：


// module.js
export var port = 3000
export function getAccounts(url) {
  ...
}

main.js中使用import导入module.js，可以指定需要导入的变量：


// main.js
import {port, getAccounts} from 'module'
console.log(port) // 输出3000

也可以将全部变量导入：


// main.js
import * as service from 'module'
console.log(service.port) // 3000
