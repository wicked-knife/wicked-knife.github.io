# 模块热替换

模块热替换是指项目开发时当有模块改动的时候，浏览器无需刷新即可自动替换更新的模块。

模块热替换只适用于开发环境。

模块热替换可以应用于CSS和js模块的热替换。

在webpack.common.js中作如下配置:

const webpack = require('webpack');
module.exports = {
  ...
    devServer: {
      ...
      hot: true
      ...
    }
    plugins: [
      new webpack.HotModuleReplacementPlugin()
    ]
  ...
}

即可实现CSS文件的热替换，当改变CSS文件内容时，浏览器无需刷新即可更新页面样式。

js模块的热替换较复杂，需要自己编写相应的逻辑且应用场景不多，这里不做赘述。具体可参看官网。
