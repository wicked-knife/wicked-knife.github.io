# 项目初始化

本笔记着重记录分析webpack中的一些复杂概念，对于一些简单配置不做讲解。

# (0) 项目初始化

使用npm快速初始化package.json文件。

npm init -y

安装webpack, webpack-cli。(webpack-cli主要用作本地项目中在终端能够直接使用webpack命令)

npm i -D webpack webpack-cli

# (1) 创建配置文件

创建webpack.common.js文件，用作webpack配置文件的公共配置。并进行如下配置：

    module.exports : {
        entry: {
            index: './index.js'
        }
    }

意为将当前目录下index.js文件用作webpack打包的入口，webpack会从文件入口开始分析模块依赖（webpack中万物皆模块），并生成依赖图。 使用对象键值对的形式进行配置打包后会自动根据键名生成打包好的文件。这样配置比较直观清晰。

在命令行中执行

webpack --config webpack.common.js

即可执行打包操作。--config参数用作指定webpack配置文件。默认为webpack.config.js。

# (2) 配置npm scripts

在项目package.json文件scripts字段进行如下配置:

{
    ...
    "scripts":{
        "bundle": "webpack --config webpack.common.js"
    }
    ...
}

在命令行中直接执行npm run bundle即可进行打包操作。
