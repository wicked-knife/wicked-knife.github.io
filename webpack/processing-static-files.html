# 处理静态资源文件

一些静态资源文件如图片，字体文件在 webpack 中也被视为模块。同样需要相应的 loader 对其进行打包。

# 图片文件

安装url-loader

npm i -D url-loader

在webpack.common.js中作如下配置:

{
  ...
  module: {
    rules: [
      test: /\.(svg|jpe?g|gif|png|bmp)$/,
      use: {
        loader: 'url-loader',
        options: {
          name: '[name]_[hash:5].[ext]',
          outputPath: 'images',
          limit: 2048
        }
      }
    ]
  }
  ...
}

可以在JS文件中这样引入图片文件:

import logo from './logo.png'

拿到的logo即是正确的图片路径。

name: 配置打包后的图片文件名。 outputPath: 配置打包后图片存放的文件夹。这样配置打包后图片都会放在dist/images文件夹下。 limit：配置对图片文件打包时的尺寸限制。大于该尺寸图片文件会被复制到dist/images目录下，小于该尺寸的图片会被打包成base64格式的字符串。

# 字体及其他静态资源文件

安装file-loader

npm i -D file-loader

在webpack.common.js中作如下配置:

{
  ...
  module: {
    rules: [
        test: /\.(eot|ttf|woff2?|otf)$/,
        use: {
          loader: 'file-loader',
          options: {
            name: '[name]_[hash:5].[ext]',
            outputPath: 'fonts'
          }
        }
    ]
  }
  ...
}

这样配置即可将字体文件打包至dist/fonts文件夹下。

其他类型的文件使用file-loader进行类似配置即可。

file-loader和url-loader功能类似，不过url-loader支持将图片转换成base64格式，所以其更适合打包图片类型文件。
