# 处理JavaScript文件

webpack 默认支持JavaScript文件文件的模块化。即可以直接使用commonjs或ESModule的语法进行JS模块的导入导出。

# ES6+语法编译

很多时候项目需要使用ES6的语法。如箭头函数。webpack 默认只支持JS文件的模块化，但是不会转化ES6的语法，为了能够使项目能够在各大浏览器上运行，有必要将ES6语法编译成ES5的语法。

使用babel进行ES6到ES5的编译。

npm i -D babel-loader @babel/core @babel/preset-env webpack

在webpack.common.js中作如下配置:

{
  ...
  module: {
    rules: [
      test: /\.js$/,
      exclude: /node_modules/
      use: {
        loader: 'babel-loader'
      }
    ]
  }
  ...
}

exclude: 指不对node_modules目录下的js文件进行编译

创建.babelrc文件对babel进行配置

module.exports = {
  	"presets": [
		[
			"@babel/preset-env"
		]
	]
}

这样配置用来指定ES6编译后的运行环境。

# ES6语法polyfill

通过polyfill我们可以放心使用ES6中新增的特性。如Promise, WeakMap以及数组或对象上的一些扩展方法。

安装babel-polyfill

npm i -d @babel-polyfill core-js

在.babelrc进行如下配置:

{
	"presets": [
		[
			"@babel/preset-env",
			{
				"useBuiltIns": "usage",
				"corejs": 3
			}
		]
	]
}

useBuiltIns: 指只对项目中使用到的 ES6 新特性做 polyfill ，这样可以大大减少打包后的代码体积。 corejs：指定corejs的版本。不配置此字段打包会报错，以后可能会优化。
