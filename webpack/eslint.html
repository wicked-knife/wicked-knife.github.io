# 在webpack中使用eslint

# 安装eslint

npm i -d eslint

# 初始化eslint配置文件

npx eslint --init

通过回答问题的形式自动生成eslint的配置文件。

# 在webpack中配置eslint

安装eslint-loader

npm i -D eslint-loader

module.exports = {
  ...
  module: {
    rules: [
      test: /\.js$/,
      exclude: /node_modules/,
      use: [
        'babel-loader',
        {
          loader: 'eslint-loader',
          options: {
            fix: true
          }
        }
      ]
    ]
  }
}

'eslint-loader' 先于 'babel-loader' 执行是因为我们直接书写的源代码需要经过eslint的语法检查，而经过 babel 编译后的js是不需要也没有必要进行语法检查的。即 eslint 只对我们直接书写的源代码进行语法检查。

fix: 代表让 eslint 自动修复一些 eslint 可以自动修复的错误。如缩进、分号。

# 配置babel-eslint

如果不配置babel-eslint,当在项目中使用动态引入,如:

import(/* webpackChunkName: "lodash" */ 'lodash').then(({ default:　_ }) => {
  // you can use lodash as _
  console.log(_)
})

eslint不识别动态引入的语法，导致eslint报错。

安装:

npm i -D babel-eslint
``

配置:

```js
// .eslintrc.js

module.exports = {
  ...
    "parser": "babel-eslint",
  ...
}

即可解决上述问题

# 自定义eslint规则

直接配置.babelrc.js文件即可。具体规则见eslint官网。
