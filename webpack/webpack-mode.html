# 开发环境的区分

webpack4 mode 有 'development' 和 'production' 两种模式，以区分开发环境和生产环境。

所以我们可以创建三个配置文件，分别用来保存公用配置，开发环境配置，生产环境配置。

安装 'webpack-merge' 用于合并配置文件。

npm i -D webpack-merge

# 开发环境

我们先列出开发环境下我们的需求:

  • 项目开发体验尽量轻松。

  • 代码调错方便

  • 能够代理http请求

  • HMR

  • 不需要代码压缩

基于上述需求我们可以将开发环境下的webpack配置文件拆分出来。

创建webpack.dev.js

const webpack = require('webpack')
const devConf = {
  mode: 'development',
  devtool: 'cheap-module-eval-source-map',
    devServer: {
    contentBase: 'dist',
    open: true,
    hot: true,
    overlay: true,
    proxy: {
      ...
    }
  },
  module: {
    rules: [
      {
        test: /\.module\.(css|scss)$/,
        use: [
          'style-loader',
          {
            loader: 'css-loader',
            options: {
              importLoaders: 2
            }
          },
          'sass-loader',
          'postcss-loader'
        ],
        test: /\.(css|scss)$/,
        use: [
          'style-loader',
          {
            loader: 'css-loader',
            options: {
              importLoaders: 2
            }
          },
          'sass-loader',
          'postcss-loader'
        ],
      }
    ]
  },
  plugins: [
    new webpack.HotModuleReplacementPlugin()
  ]
}

根据上述需求将相应配置抽离出来。

# 生产环境

我们先列出开发环境下我们的需求:

  • JS压缩， tree-shaking

  • 线上代码出错时能够快速定位。

  • CSS文件拆分，压缩

基于上述需求我们可以将开发环境下的webpack配置文件拆分出来。

创建webpack.prod.js

const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const OptimizeCSSAssetsPlugin = require('optimize-css-assets-webpack-plugin')
const devConf = {
  mode: 'development',
  devtool: 'cheap-module-eval-source-map',
    devServer: {
    contentBase: 'dist',
    open: true,
    hot: true,
    overlay: true,
    proxy: {
      ...
    }
  },
  module: {
    rules: [
      {
        test: /\.module\.(css|scss)$/,
        use: [
          MiniCssExtractPlugin.loader,
          {
            loader: 'css-loader',
            options: {
              importLoaders: 2
            }
          },
          'sass-loader',
          'postcss-loader'
        ],
        test: /\.(css|scss)$/,
        use: [
          MiniCssExtractPlugin.loader,
          {
            loader: 'css-loader',
            options: {
              importLoaders: 2
            }
          },
          'sass-loader',
          'postcss-loader'
        ],
      }
    ]
  },
  plugins: [
     new MiniCssExtractPlugin()
  ],
  optimization: {
    minimizer: [
      new OptimizeCSSAssetsPlugin({})
    ]
  }
}

根据上述需求将相应配置抽离出来。

# 公用配置

公用配置是开发和生产环境公用的代码。

比如entry, output等配置。

创建webpack.common.js

const HtmlWebpackPlugin = require('html-webpack-plugin')
const CleanWebpackPlugin = require('clean-webpack-plugin')
module.exports = {
  entry: {
    index: './src/index.js',
    list: './src/list.js'
  },
  output: {
    filename: '[name]_[hash:5].js',
    chunkFilename: '[name]_[contenthash:5].js'
  },
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: [
          'babel-loader',
          {
            loader: 'eslint-loader',
            options: {
              fix: true
            }
          }
        ]
      },
      {
        test: /\.(svg|jpe?g|gif|png|bmp)$/,
        use: {
          loader: 'url-loader',
          options: {
            name: '[name]_[hash:5].[ext]',
            outputPath: 'images',
            limit: 2048
          }
        }
      },
      {
        test: /\.(eot|ttf|woff2?|otf)$/,
        use: {
          loader: 'file-loader',
          options: {
            name: '[name]_[hash:5].[ext]',
            outputPath: 'fonts'
          }
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin(),
    new CleanWebpackPlugin()
  ]
}

使用'webpack-merge'对配置文件进行合并。

// webpack.dev.js
const merge = require('webpack-merge')
const commonConf = require('./webpack.common.js')
....
module.exports = merge(commonConf, devConf)

// webpack.prod.js
const merge = require('webpack-merge')
const commonConf = require('./webpack.common.js')
....
module.exports = merge(commonConf, prodConf)

最后在package.json 配置 npm scripts

{
  ...
  "scripts": {
    "dev": "webpack-dev-server --config webpack.dev.js",
    "build": "webpack --config webpack.prod.js"
  }
  ...
}

通过npm run dev和npm run build来执行项目的开发和打包。
