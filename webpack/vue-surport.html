# 添加 Vue 支持

当基于 Vue 进行项目开发时，我们通常需要编写.vue形式的单文件组件。通过使用vue-loader我们可以在 webpack 中解析.vue文件。

安装vue-loader 和 vue-template-compiler

根据 vue-loader 官方文档：

你每次升级项目中的 vue 包时，也应该匹配升级 vue-template-compiler。

npm i -D vue-loader vue-template-compiler

在webpack.common.js中作如下配置:

const VueLoaderPlugin = require('vue-loader/lib/plugin')
module.exports = {
    ...
    modules: {
      rules: [
        ...
        {
          test: /\.vue$/,
          loader: 'vue-loader'
        }
        ...
      ]
    },
    plugins: [
      ...
      new VueLoaderPlugin()
    ]
}

即可在项目中直接把.vue文件当做模块进行引入。

这样配置在开发环境下是没有问题的，但在生产环境下进行打包的话会发现没有在.vue文件中书写的 css 没有打包出来。

通过查阅官方文档:

如果你在开发一个库或多项目仓库 (monorepo)，请注意导入 CSS 是具有副作用的。
请确保在 package.json 中移除 "sideEffects": false，
否则 CSS 代码块会在生产环境构建时被 webpack 丢掉。

但是我们有时候是需要引入一些带有sideEffects的模块的，比如 CSS 重置,babel-polyfill。所以我们应该进行这样的配置:

	"sideEffects": [
		"@babel/polyfill",
		"*.css",
       "*.scss",
       "*.vue"
	]

这样将所有的.vue文件标记为sideEffects模块，这样.vue文件中的 CSS 代码就可以正确打包了。

在.vue文件中使用import(...)动态加载模块时，eslint 也许会警报，在.eslintrc.js中进行如下配置:

    'parserOptions': {
+++     'parser': 'babel-eslint',
        'ecmaVersion': 2018,
        'sourceType': 'module'
    },
